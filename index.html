給紅蟲的操作指南

建立個人外部知識庫 · 概念參考 Karpathy 的 LLM Wiki · 用 Claude Desktop 內建的 Claude Code + Obsidian

先搞懂一件事：這不是傳統 RAG

與其每次都重新去撈原始文件，不如讓 Claude 增量維護一個「會複利」的 markdown 知識庫——知識整理一次就長在裡面，交叉連結持續累積。你負責丟資料、問好問題；Claude 負責所有繁瑣的整理與交叉引用。整個庫分成三層：

原始資料

你丟進來的東西，Claude 只讀不改

整理頁

Claude 產生與維護的摘要、概念頁

CLAUDE.md 規範

定義結構與工作流，這份讓 Claude Code 自己寫

順序的心法：知識庫本質上就是「一個 Claude 維護的資料夾」，所以先讓 Claude Code 把資料夾建好，Obsidian 只是事後拿來瀏覽它的工具。

1

開 Claude Code，指定一個資料夾當知識庫

你已經裝好的 Claude Desktop 內建 Claude Code——它跟終端機的 CLI 版是同一個 agent，會自動讀資料夾裡的 CLAUDE.md、也能直接讀寫檔案，只是介面是圖形化的。

  • 打開 Claude Desktop，開啟 Claude Code 面板。
  • 新建一個空資料夾（例如 my-kb），把它設成工作資料夾（working directory）——這個資料夾就是你的知識庫本體。
2

用一句話，讓 Claude Code 用「問答」陪你把庫建起來

不用預先想好結構，也不用自己寫 CLAUDE.md。把下面這段貼給 Claude Code，它會先反過來問你用途，再依你的回答逐步建好目錄結構與規範檔。（想了解背後概念，可看 Karpathy 的 LLM wiki 原文）

我想在這個資料夾建立一個「個人外部知識庫」，概念參考 Andrej Karpathy 的 LLM wiki：讓你增量維護一個會複利的 markdown 知識庫，而不是每次重查的 RAG。

我還沒想好完整結構，想跟你一起逐步建立。請先用問句了解我的用途——例如我主要想收什麼類型的內容、平常會怎麼用它、想解決什麼問題——一次問一個問題。等你了解得差不多，再依我的回答幫我設計並建立：
- 合適的資料夾與檔案結構（原始資料、你維護的整理頁、index、log 等）
- 一份放在根目錄的 CLAUDE.md 當規範，定義檔名、frontmatter、用 [[wikilinks]] 交叉連結，以及日常的 ingest / query / lint 工作流

全部用 markdown、連結一律用 Obsidian 的 [[wikilink]] 語法。先問我第一個問題吧。
3

裝 Obsidian，把這個資料夾打開成 vault

Obsidian 是一個「打開資料夾當筆記本」的免費 app，原生支援 markdown 和 [[wikilink]]，拿來瀏覽 Claude 建好的知識庫剛剛好。

  • 到 obsidian.md/download 下載並安裝（Mac／Windows 皆可，免費）。
  • 打開後選 Open folder as vault，選第 1 步那個資料夾——Claude 建的頁面就全出現了。
4

之後就自然地用它

建好之後不需要背任何特殊指令——因為 CLAUDE.md 已經讓 Claude Code 知道這個知識庫的規範與存在，你直接用日常語言講「幫我把這篇收進來」「根據我的庫回答某問題」就好。你也不必每次都待在同一個面板——只要 Claude Code 讀得到那個資料夾，在哪跟它講都行。回到 Obsidian，Claude 寫的頁面會即時出現；點 [[wikilink]] 在頁面間跳轉，打開 Graph view（左側工具列的關係圖示）就能看到知識庫的連結網絡慢慢長大——這就是「複利」的樣子。

心法：人負責「丟好的資料、問好的問題」，Claude 負責所有 bookkeeping（交叉引用、一致性、一次改很多頁）。庫越用越值錢。
